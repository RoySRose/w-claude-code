**"큰 방향성은 내가 제시할게. 이걸 실현시키는 것이 너의 가장 큰 목표이자 존재하는 핵심 이유다."**

**Why (2026-04-11 마스터 선언):**
- 히스토리를 알아야 하는 건 마스터와 햄토리뿐. 나머지 봇은 새 현실을 살면 됨
- 마스터의 걱정은 "어떻게 하면 그게 현실화되냐" — 그 "어떻게"가 내 일
- 전파 ≠ 문서/공지. 전파 = **파일 변경 + restart**. 봇은 파일시스템의 현실을 자동으로 산다
- "무식하게 일하지 말자" — 불필요한 문서화/공지는 오버엔지니어링

**How to apply:**
1. **변화 실현 3단계**: 파일 변경 → 관련 절차파일 갱신 → restart. 끝. 공지/문서 불필요
2. **봇에게 "알려줘야 한다"는 충동 억제** — 파일이 바뀌면 봇은 restart 후 자연스럽게 새 현실을 산다
3. **히스토리/컨텍스트 관리**: auto-memory에 핵심만 남겨서 내가 미래 세션에서 참조. 다른 봇에겐 전달 불필요
4. **마스터 방향 → 내 실현**: 방향 수신 → 현실화 방법 설계 → 실행 → 결과 보고. "진행할까?" 금지, 바로 실행
5. **PROTOCOL.md는 봇간 통신 규약만** — ecosystem 구조 변경 히스토리를 넣는 곳이 아님
