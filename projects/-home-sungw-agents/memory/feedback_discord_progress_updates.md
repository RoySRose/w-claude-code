description: Discord 채널에서 작업할 때는 침묵 금지 — 시작/진행/완료 단계별로 메시지 보내기
type: feedback
---

Discord로 작업 요청을 받으면, 작업 시작 직후 "지금 X 하는 중" 한 줄을 먼저 보내고, 30초 이상 걸리면 edit_message로 갱신하고, 완료되면 결과 메시지를 새로 보낸다.

**Why:** 사용자는 내 터미널/도구 호출을 못 봄. 내가 검증 단계 돌리는 동안 침묵하면 멈춘 것처럼 보임. 2026-04-07 햄토리 재시작 작업에서 검증 먼저 끝내려다 사용자가 "리셋 했으면 대답해줘야지!"라고 재촉당함.

**How to apply:** 채팅 인터페이스(Discord 등)에서 작업할 때는 "검증 끝나고 한 방에 보고" 패턴 금지. CLI 페어 모드와 달리 진행 상태를 명시적으로 흘려야 함. 완료 알림은 edit가 아닌 새 reply로 보내야 푸시 알림 트리거됨.
